# 部署工具 使用说明

## 一、简介

部署工具是一款 Java 微服务一键部署桌面客户端，通过图形界面完成本地 Maven 打包 → SFTP 上传 Jar → SSH 执行启动脚本的全流程。

---

## 二、快速开始

### 1. 添加服务器

点击工具栏 **「+ 服务器」**，填写以下信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| 服务器名称 | 给服务器起个名字 | 测试环境服务器 |
| IP 地址 | 服务器 IP | 192.168.90.16 |
| 用户名 | SSH 登录用户名 | root |
| 密码 | SSH 登录密码 | —— |
| 项目路径 | 服务器上项目所在目录 | /home/wuyuan/server/jingzhu-imaster |
| 备注 | 备注信息（可选） | —— |

> ⚠️ **密码不会被泄露**，在配置文件中以 AES 加密存储。

---

### 2. 添加应用

在左侧树形列表中选中一个服务器，点击 **「+ 应用」**，填写以下信息：

| 字段 | 说明 | 示例 |
|---|---|---|
| 应用名称 | 应用标识名 | industry |
| Jar 包名 | 编译后的 Jar 包名 | industry-0.0.1-SNAPSHOT.jar |
| 启动脚本 | 服务器上的 .sh 脚本名 | uu-industry.sh |
| 脚本路径 | 脚本所在目录（不填则使用项目路径） | /home/wuyuan/server/scripts |
| 脚本参数 | start / stop / restart | restart |
| Maven模块 | 多模块项目时填子模块名（可选） | industry-module |
| 本地项目路径 | 本地 IDEA 项目根目录 | D:/ideaProjects/companyProject/IMasterR/iMaster-Api |

> 💡 **脚本路径**：如果启动脚本 `uu-industry.sh` 和 Jar 包不在同一个目录，在这里填写脚本所在的目录路径。

---

### 3. 一键部署

1. 在左侧列表选中要部署的应用
2. 点击右侧 **「⚡ 部署」** 按钮
3. 等待 6 步自动完成：
   - [1/6] 📦 Maven 打包
   - [2/6] 📦 查找 Jar 包
   - [3/6] 📤 上传 Jar 到服务器
   - [4/6] 📝 生成部署记录（deploy.txt）
   - [5/6] 🔍 验证服务器文件
   - [6/6] 🖥️ 执行启动脚本

---

## 三、全局设置

点击工具栏 **「⚙ 设置」**，可以配置：

| 设置项 | 说明 |
|---|---|
| JDK 路径 | 本机 JDK 安装路径（不填则用系统 PATH 中的 java） |
| Maven 路径 | 本机 Maven 安装路径（不填则用系统 PATH 中的 mvn） |
| Maven 仓库 | 本机 Maven 仓库路径（不填则用默认 .m2/repository） |
| 私有仓库认证 | 如果 Maven 仓库需要账号密码，在此配置 |

---

## 四、数据存放

- **数据库路径**：`D:\skills\deply\data\deploy_tool.db`
- **配置文件**：随数据库路径存放，升级软件不丢失数据
- **部署记录**：每次部署后会在服务器项目目录下生成 `deploy.txt`

## 五、常见问题

### 脚本执行成功但 Java 没启动？

SSH 默认非登录 shell，PATH 中没有 java 命令。工具已通过 `bash -l -c` 加载登录 shell 环境解决这个问题。

### 部署时脚本找不到？

确认「脚本路径」填写正确（如果脚本不和 Jar 在同级目录），或者确认「启动脚本」名称和服务器上的文件名完全一致。

### Maven 打包失败？

确认本地项目路径正确，JDK 和 Maven 配置完整，且项目能通过 `mvn clean package` 正常编译。
